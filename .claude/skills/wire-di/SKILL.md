Wire 依赖注入快速参考 Skill。理解和使用 Wire 进行依赖注入。

## 基本概念

Wire 是一个代码生成器，用于自动编写依赖注入代码。

```go
//go:generate wire
```

运行 `wire` 命令生成 `wire_gen.go` 文件。

## 核心模式

### 1. ProviderSet 模式

每个包定义自己的 ProviderSet：

```go
// internal/apiserver/store/store.go

var ProviderSet = wire.NewSet(NewStore, wire.Bind(new(IStore), new(*datastore)))

// NewStore 是构造函数
func NewStore(db *gorm.DB) *datastore {
    return &datastore{db}
}
```

```go
// internal/apiserver/biz/biz.go

var ProviderSet = wire.NewSet(NewBiz, wire.Bind(new(IBiz), new(*biz)))

func NewBiz(store store.IStore, authz *authz.Authz) *biz {
    return &biz{store: store, authz: authz}
}
```

### 2. 接口绑定模式

使用 `wire.Bind` 将接口与实现绑定：

```go
// 绑定 IStore 接口到 *datastore 实现
wire.Bind(new(IStore), new(*datastore))

// 绑定 IBiz 接口到 *biz 实现
wire.Bind(new(IBiz), new(*biz))
```

### 3. 结构体字段注入

```go
// 注入结构体的所有字段
wire.Struct(new(ServerConfig), "*")

// 注入特定字段
wire.Struct(new(ServerConfig), "DB", "Config")
```

### 4. 字段提取

```go
// 从 Config 中提取 ServerMode 字段
wire.FieldsOf(new(*Config), "ServerMode")
```

## 完整示例

### wire.go 文件

```go
//go:build wireinject
// +build wireinject

package apiserver

import (
    "github.com/google/wire"
    "github.com/onexstack/onexstack/pkg/authz"

    "github.com/ashwinyue/eino-show/internal/apiserver/biz"
    "github.com/ashwinyue/eino-show/internal/apiserver/store"
    "github.com/ashwinyue/eino-show/internal/apiserver/pkg/validation"
)

func InitializeWebServer(cfg *Config) (server.Server, error) {
    wire.Build(
        // 服务器初始化
        wire.NewSet(NewWebServer, wire.FieldsOf(new(*Config), "ServerMode")),
        wire.Struct(new(ServerConfig), "*"),

        // 数据层
        wire.NewSet(store.ProviderSet, ProvideDB),

        // 业务层
        wire.NewSet(biz.ProviderSet),

        // 验证层
        validation.ProviderSet,

        // 授权
        authz.ProviderSet,
    )
    return nil, nil
}
```

### 生成后的 wire_gen.go

```go
// Code generated by Wire. DO NOT EDIT.

func InitializeWebServer(cfg *Config) (server.Server, error) {
    db := ProvideDB(cfg)
    datastore := store.NewStore(db)
    istore := datastore
    bizInstance := biz.NewBiz(istore, authz.NewAuthz())
    ibiz := bizInstance
    serverConfig := &ServerConfig{...}
    webServer := NewWebServer(serverConfig, ibiz)
    return webServer, nil
}
```

## 新模块添加流程

添加新的业务模块（如 Session）时：

### 1. Store 层

```go
// internal/apiserver/store/session.go

package store

// 接口定义
type SessionStore interface { ... }

// 实现
type sessionStore struct { ... }

func newSessionStore(store *datastore) *sessionStore {
    return &sessionStore{...}
}

// 在 IStore 中添加方法
func (s *datastore) Session() SessionStore {
    return newSessionStore(s)
}
```

### 2. Biz 层

```go
// internal/apiserver/biz/v1/session/session.go

package session

// 接口定义
type SessionBiz interface { ... }

// 实现
type biz struct {
    store store.IStore
}

func New(store store.IStore) SessionBiz {
    return &biz{store: store}
}
```

```go
// internal/apiserver/biz/biz.go

func (b *biz) Session() sessionbiz.SessionBiz {
    return sessionbiz.New(b.store)
}
```

### 3. 更新 ProviderSet（如需要）

```go
// internal/apiserver/biz/v1/session/session.go

var ProviderSet = wire.NewSet(New)
```

## Wire 函数签名规则

| 函数签名 | Wire 行为 |
|---------|-----------|
| `func NewFoo() *Foo` | 提供 *Foo 类型 |
| `func NewFoo() Foo` | 提供 Foo 类型 |
| `func NewFoo() IFoo` | 提供 IFoo 接口 |
| `func NewFoo(bar *Bar)` | 依赖 *Bar，提供 *Foo |
| `func NewFoo(cfg *Config)` | 依赖 *Config |

## 常见错误

### 错误 1：忘记绑定接口

```go
// 错误：wire 无法知道 *biz 实现了 IBiz
type IBiz interface { ... }
type biz struct { ... }

// 正确：使用 wire.Bind
wire.Bind(new(IBiz), new(*biz))
```

### 错误 2：ProviderSet 未导入

```go
// 错误：忘记添加 biz.ProviderSet
wire.Build(
    store.ProviderSet,
    // biz.ProviderSet 缺失
)

// 正确
wire.Build(
    store.ProviderSet,
    biz.ProviderSet,
)
```

### 错误 3：循环依赖

```go
// 错误：A 依赖 B，B 依赖 A
func NewA(b *B) *A
func NewB(a *A) *B

// 解决：通过接口解耦或重新设计
```

## 生成和验证

```bash
# 生成 Wire 代码
wire ./internal/apiserver

# 验证生成结果
cat internal/apiserver/wire_gen.go

# 如果遇到错误，检查：
# 1. //go:build wireinject 标签是否存在
# 2. ProviderSet 是否正确导入
# 3. 接口绑定是否正确
```
