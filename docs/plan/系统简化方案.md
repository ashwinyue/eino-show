# 系统简化方案

> 将复杂的 eino-show 系统简化为专注于对话的小程序应用

## 背景

当前系统功能过于复杂，包含：
- 数据库驱动的动态 Agent 配置
- 完整的 RAG 知识库系统
- MCP 协议支持
- 复杂的多 Agent 编排（Supervisor/Workflow）
- 多租户管理
- 完整的前端配置页面

**简化目标**：专注于对话体验，打造一个轻量级的小程序对话应用。

---

## 简化目标

| 方面 | 当前 | 目标 |
|------|------|------|
| Agent 配置 | 数据库动态配置 | YAML 固定配置 |
| 知识库 | 完整 RAG 系统 | 无需 |
| 前端页面 | 多页面（配置/知识库/Agent） | 单一对话页面 |
| Agent 模式 | 多种复杂模式 | 主 Agent + 子 Agent |
| 触发方式 | 用户选择 | 对话自动触发 |

## 配置文件设计

### `configs/agents.yaml`

```yaml
# 主 Agent 配置
main_agent:
  name: "智能助手"
  model: "gpt-4o"
  system_prompt: |
    你是一个智能助手，可以自动识别用户意图并转发给对应的专家助手。
    可用的专家助手包括：编码助手、搜索助手、通用助手。

  # 意图识别规则
  intent_rules:
    coder:
      keywords: ["代码", "编程", "函数", "bug", "API", "开发"]
      description: "编程和代码相关问题"

    searcher:
      keywords: ["搜索", "查询", "最新", "新闻", "天气"]
      description: "信息检索和实时查询"

    general:
      keywords: []
      description: "通用对话"

# 子 Agent 配置
sub_agents:
  coder:
    name: "编码助手"
    model: "claude-3-5-sonnet"
    system_prompt: |
      你是编程专家，擅长：
      - 代码编写和调试
      - API 设计和调用
      - 技术问题解答
      请提供简洁准确的代码示例。

    tools:
      - web_search

  searcher:
    name: "搜索助手"
    model: "gpt-4o"
    system_prompt: |
      你是搜索专家，擅长：
      - 实时信息检索
      - 多源信息整合
      - 事实核查

    tools:
      - web_search

  general:
    name: "通用助手"
    model: "gpt-4o"
    system_prompt: |
      你是一个友好、专业的对话助手。
    tools: []
```

## 实施步骤

### 第一阶段：清理冗余模块

1. 删除 RAG 相关所有文件
2. 删除 MCP 相关所有文件
3. 删除复杂的 Agent 管理逻辑
4. 删除前端配置页面

### 第二阶段：简化核心模块

1. Agent 配置改为 YAML 固定配置
2. 简化 Agent 工厂，移除数据库配置
3. 保留核心 SSE 流式输出
4. 简化 Session 存储

### 第三阶段：实现主 Agent

1. 实现基于关键词的意图识别
2. 实现 Agent 自动转发逻辑
3. 优化流式输出体验

### 第四阶段：前端简化

1. 只保留单一对话页面
2. 移除复杂组件
3. 优化移动端体验
