# eino-show 重构计划

## 重构目标

1. 采用 **miniblog-x 脚手架**风格重构代码结构
2. 集成 **Eino ADK** 作为 Agent 框架
3. **保持接口不变** - 前端无需修改
4. **保持数据表设计不变** - 平滑迁移
5. 先实现核心功能，外部扩展（ES、知识图谱）暂缓

---

## 核心功能范围

### ✅ 本次重构包含

| 模块 | 功能 | 说明 |
|------|------|------|
| **Agent** | ReAct 循环执行 | 基于 Eino ReactAgent |
| **Agent** | 流式输出 (SSE) | 事件总线机制 |
| **Agent** | 内置智能体 | QuickAnswer、SmartReasoning |
| **Agent** | 工具调用 | knowledge_search、grep_chunks、web_search |
| **Session** | 会话管理 | CRUD + 消息历史 |
| **Session** | 上下文管理 | 渐进式压缩 |
| **Knowledge** | 知识库管理 | CRUD + 统计 |
| **Knowledge** | 向量搜索 | PGVector 语义搜索 |

| **User** | 用户管理 | 基本 CRUD |
| **Auth** | JWT 认证 | 基于 Tenant |

**Agent 接口定义**:
```go
// internal/pkg/agent/agent.go
package agent

import "context"

type Agent interface {
    // Run 同步执行 Agent，返回完整结果
    Run(ctx context.Context, input *Input) (*Output, error)

    // StreamRun 流式执行 Agent，通过事件通道返回
    StreamRun(ctx context.Context, input *Input) (<-chan *Event, error)
}

type Input struct {
    Query     string                 // 用户查询
    SessionID string                 // 会话ID
    Variables map[string]interface{} // 变量
    Tools     []string               // 可用工具列表
}

type Output struct {
    Answer   string                 // 最终答案
    Steps    []*Step                // 执行步骤
    Metadata map[string]interface{} // 元数据
}

type Event struct {
    Type EventType
    Data interface{}
}

type EventType string

const (
    EventTypeThinking   EventType = "agent_thinking"    // 思考中
    EventTypeAction     EventType = "agent_action"      // 工具调用
    EventTypeObservation EventType = "agent_observation" // 工具结果
    EventTypeComplete   EventType = "agent_complete"    // 完成
    EventTypeError      EventType = "agent_error"       // 错误
)
```
