# eino-show 重构计划

## 重构目标

1. 采用 **miniblog-x 脚手架**风格重构代码结构
2. 集成 **Eino ADK** 作为 Agent 框架
3. **保持接口不变** - 前端无需修改
4. **保持数据表设计不变** - 平滑迁移
5. 先实现核心功能，外部扩展（ES、知识图谱）暂缓

---

## 核心功能范围

### ✅ 本次重构包含

| 模块 | 功能 | 说明 |
|------|------|------|
| **Agent** | ReAct 循环执行 | 基于 Eino ReactAgent |
| **Agent** | 流式输出 (SSE) | 事件总线机制 |
| **Agent** | 内置智能体 | QuickAnswer、SmartReasoning |
| **Agent** | 工具调用 | knowledge_search、grep_chunks、web_search |
| **Session** | 会话管理 | CRUD + 消息历史 |
| **Session** | 上下文管理 | 渐进式压缩 |
| **Knowledge** | 知识库管理 | CRUD + 统计 |
| **Knowledge** | 向量搜索 | PGVector 语义搜索 |
| **Knowledge** | 文档上传/解析 | 分块存储 |
| **User** | 用户管理 | 基本 CRUD |
| **Auth** | JWT 认证 | 基于 Tenant |

### ⏸️ 暂缓实现

| 模块 | 功能 | 暂缓原因 |
|------|------|---------|
| **Knowledge** | Elasticsearch | 后续版本 |
| **Knowledge** | 知识图谱查询 | 后续版本 |
| **Agent** | MCP 集成 | 需要先定义规范 |
| **Agent** | 数据分析工具 | 需要数据库设计 |

## 架构决策记录

### ADR-001: 直接使用 Eino 框架

**日期**: 2026-01-16

**状态**: 已采纳

**背景**:
- Eino 已提供完善的 Agent 抽象
- 自定义抽象层会增加复杂度和维护成本

**决策**:
- 直接使用 Eino 的 `Agent` 和 `Tool` 接口
- 仅在业务层（Biz）做必要的适配
- SSE 事件格式在 HTTP 层处理

**后果**:
- ✅ 代码更简洁
- ✅ 跟随 Eino 社区演进
- ⚠️ 未来替换框架成本较高
